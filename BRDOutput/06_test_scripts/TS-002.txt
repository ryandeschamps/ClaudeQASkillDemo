Test Script: TS-002 - User Login with Email and Password

Test Scenario: As a registered buyer, I want to login using my email and password, so that I can access my account and make purchases.

Prerequisites:
- User has a registered and verified account
- Website is accessible
- User is not currently logged in

Test Data:
- Email: john.doe@example.com
- Password: Test@1234

Test Steps:

GIVEN the user is on the website homepage
AND the user is not logged in
WHEN the user clicks on the "Login" or "Sign In" button
THEN the login form should be displayed
AND the form should have fields for Email and Password
AND the form should have a "Login" button
AND the form should have a "Forgot Password" link

GIVEN the user is viewing the login form
WHEN the user enters email "john.doe@example.com" in the Email field
AND the user enters password "Test@1234" in the Password field
AND the user clicks the "Login" button
THEN the system should validate the credentials
AND the system should authenticate the user
AND the system should create a user session
AND the user should be redirected to the homepage or account dashboard
AND the user's name should be displayed in the header (e.g., "Welcome, John")
AND the "Login" button should be replaced with "My Account" and "Logout" options

GIVEN the user is logged in
WHEN the user navigates to different pages on the website
THEN the user should remain logged in
AND the session should persist across page navigations

Expected Result:
- Login form is displayed with Email and Password fields
- User enters valid credentials and clicks Login
- System validates credentials successfully
- User session is created
- User is redirected to homepage/dashboard
- User's name is displayed indicating successful login
- Navigation menu shows "My Account" and "Logout" options
- Session persists across page navigations

Validation Points:
- Login form is accessible from homepage
- Email and Password fields accept input
- System validates email format
- System authenticates correct credentials
- Session is created and maintained
- User-specific UI elements are displayed after login
- User can access authenticated features (cart, wishlist, account)
