Test Script: TS-064 - Add New Product to Catalog (Admin)

Test Scenario: As an admin user, I want to add new products with all details and variations, so that I can make them available for purchase.

Prerequisites:
- Admin user is logged into admin panel
- Product categories and sub-categories exist
- Admin has permissions to manage products
- Image files are prepared for upload

Test Data:
- Product Name: Premium Cotton Polo Shirt
- SKU: MEN-POL-001
- Description: High-quality cotton polo shirt perfect for casual and semi-formal occasions
- Category: Men's Apparel
- Sub-category: Shirts
- Base Price: $54.99
- Keywords: polo, shirt, cotton, casual, men
- Status: Active
- Sizes: S, M, L, XL, XXL
- Colors: Navy, White, Black, Red
- Images: polo_navy_front.jpg, polo_navy_back.jpg, polo_white_front.jpg

Test Steps:

GIVEN the admin user is logged into the admin panel
WHEN the admin navigates to "Product Management" section
THEN the system should display the product management dashboard
AND the dashboard should show a list of existing products
AND the dashboard should have an "Add New Product" button

GIVEN the admin is viewing the product management dashboard
WHEN the admin clicks the "Add New Product" button
THEN the system should display the add product form
AND the form should have the following fields:
  - Product Name (text input, required)
  - SKU (text input, required, unique)
  - Description (textarea, required)
  - Category (dropdown, required)
  - Sub-category (dropdown, required)
  - Base Price (number input, required)
  - Keywords (text input, optional)
  - Status (radio buttons: Active/Inactive)
  - Images (file upload, multiple)
  - Variations section (sizes and colors)

GIVEN the admin is viewing the add product form
WHEN the admin enters the following information:
  - Product Name: "Premium Cotton Polo Shirt"
  - SKU: "MEN-POL-001"
  - Description: "High-quality cotton polo shirt perfect for casual and semi-formal occasions"
  - Category: "Men's Apparel"
  - Sub-category: "Shirts"
  - Base Price: "54.99"
  - Keywords: "polo, shirt, cotton, casual, men"
AND the admin selects Status: "Active"
THEN the system should accept all valid inputs

GIVEN the admin has entered basic product information
WHEN the admin adds product variations in the Variations section:
  - Sizes: S, M, L, XL, XXL
  - Colors: Navy, White, Black, Red
THEN the system should create all size-color combinations
AND the system should display a preview of all variations (20 combinations)

GIVEN the admin has added variations
WHEN the admin uploads product images:
  - polo_navy_front.jpg (set as thumbnail)
  - polo_navy_back.jpg
  - polo_white_front.jpg
THEN the system should validate image format (JPEG, PNG, WebP)
AND the system should upload images to the server
AND the system should display image previews

GIVEN all product information has been entered
WHEN the admin clicks the "Save Product" button
THEN the system should validate all required fields
AND the system should check SKU uniqueness
AND the system should save the product to the database
AND the system should generate unique Product ID
AND the system should associate all variations with the product
AND the system should link uploaded images to the product
AND the system should display a success message: "Product added successfully!"

GIVEN the product has been saved
WHEN the admin returns to the product management dashboard
THEN the new product "Premium Cotton Polo Shirt" should appear in the product list
AND the product should show:
  - Product Name: Premium Cotton Polo Shirt
  - SKU: MEN-POL-001
  - Category: Men's Apparel > Shirts
  - Price: $54.99
  - Status: Active
  - Number of Variations: 20

GIVEN the product status is "Active"
WHEN a buyer visits the website frontend
AND the buyer navigates to Men's Apparel > Shirts category
THEN the new product "Premium Cotton Polo Shirt" should be visible in the product listing
AND the product should display the thumbnail image
AND the product should show the price $54.99

GIVEN the buyer is viewing the product listing
WHEN the buyer clicks on "Premium Cotton Polo Shirt"
THEN the product detail page should display:
  - All uploaded images
  - Product name and description
  - Price: $54.99
  - Size options: S, M, L, XL, XXL
  - Color options: Navy, White, Black, Red
  - Add to Cart button
  - Add to Wishlist button

Expected Result:
- Admin successfully accesses product management
- Add product form displays with all required fields
- Admin enters complete product information
- Admin adds multiple size and color variations
- System generates all size-color combinations
- Admin uploads multiple product images
- System validates and saves all product data
- Product is assigned a unique ID and saved to database
- All variations are created and linked to product
- Images are uploaded and associated with product
- Success message confirms product creation
- Product appears in admin product list with correct details
- Product is immediately visible on frontend (status: Active)
- Buyers can view product in category listing
- Product detail page displays all information correctly
- Buyers can select variations and add to cart

Validation Points:
- Admin panel is accessible and secure
- Product form has all necessary fields
- Required fields are enforced
- SKU uniqueness is validated
- Price accepts decimal values
- Category and sub-category dropdowns are populated
- Variation system creates all combinations correctly
- Image upload validates file types and sizes
- Product is saved with all associated data (variations, images)
- Product status controls frontend visibility
- Frontend displays product immediately when Active
- Product detail page shows all variations correctly
- Buyers can interact with the product (add to cart, wishlist)
