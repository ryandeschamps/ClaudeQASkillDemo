TEST SCRIPT: TS-033
Test Scenario: Pay with Debit Card
User Story: As a Buyer in Checkout, I want to pay for my order using a debit card, so that I can complete my purchase
Priority: Critical
Related Requirements: FR-008, FR-020

GIVEN the user is logged in
  AND the user has reviewed the order summary
  AND the user is on the payment page
  AND Stripe payment integration is configured

WHEN the user selects "Debit Card" as payment method
  AND enters valid debit card details:
    - Card number
    - Cardholder name
    - Expiry date (MM/YY)
    - CVV
  AND clicks "Pay Now" or "Complete Order"

THEN the system should:
  - Validate card details format
  - Send payment request to Stripe gateway
  - Process the payment
  - Display processing indicator
  - Create order upon successful payment
  - Display order confirmation page
  - Send order confirmation email

WHEN card details are invalid or incomplete

THEN the system should display validation error messages
  AND prevent payment submission

WHEN payment is declined by bank

THEN the system should display appropriate error message
  AND allow the user to retry or use different payment method
  AND not create the order

EXPECTED RESULT:
- Debit card payment processes successfully
- Payment gateway integration works correctly
- Order is created only after successful payment
- Confirmation email is sent
- User data and payment info are transmitted securely (SSL)
- Error handling is clear and helpful
- Transaction completes within acceptable time per NFR-002
