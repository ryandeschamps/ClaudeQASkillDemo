TEST SCRIPT: TS-022
Test Scenario: Add Product to Wishlist
User Story: As a Logged-in Buyer, I want to add products to my wishlist, so that I can save items for future consideration
Priority: Medium
Related Requirements: FR-006

GIVEN the user is logged in
  AND the user is viewing a product details page or product listing

WHEN the user clicks the "Add to Wishlist" button/icon

THEN the system should:
  - Add the product to the user's wishlist
  - Display a confirmation message (e.g., "Added to wishlist")
  - Update the wishlist icon to indicate item is wishlisted
  - Update wishlist count in the header (if displayed)

WHEN the user tries to add a product already in the wishlist

THEN the system should:
  - Display a message "Already in wishlist" or similar
  - Not create a duplicate entry
  - Optionally offer to view the wishlist

WHEN the user is not logged in and clicks "Add to Wishlist"

THEN the system should redirect to the login page
  AND after login, complete the wishlist addition

EXPECTED RESULT:
- Products are added to wishlist successfully
- No duplicate entries are created
- Wishlist count updates correctly
- Visual feedback is immediate and clear
- Login is required for wishlist functionality
- Wishlist persists across sessions
