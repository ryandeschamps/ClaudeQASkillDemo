TEST SCRIPT: TS-049
Test Scenario: Write Product Review
User Story: As a Buyer who purchased a product, I want to write a detailed review of a product, so that I can share my experience with other shoppers
Priority: Medium
Related Requirements: FR-011

GIVEN the user is logged in
  AND the user has purchased and received a specific product
  AND the user is on the product details page or order history

WHEN the user clicks "Write a Review" button
  AND the review form is displayed

THEN the system should allow the user to:
  - Select a star rating (1-5 stars)
  - Enter a review title
  - Write detailed review text
  - Optionally upload photos
  - Submit the review

WHEN the user submits the review with all required fields

THEN the system should:
  - Validate all required fields are completed
  - Save the review in "Pending" status
  - Display confirmation message that review is under moderation
  - Send notification to admin for review approval

WHEN the user has not purchased the product

THEN the system should not allow posting a review
  AND display message "Purchase required to review"

WHEN the user has already reviewed the product

THEN the system should either:
  - Not show "Write Review" button, OR
  - Show "Edit Review" option instead

EXPECTED RESULT:
- Only verified purchasers can write reviews
- Review form is easy to use
- All review components are captured
- Reviews require moderation before publishing
- Duplicate reviews are prevented
- User receives confirmation of submission
- Photos can be uploaded if feature is available
