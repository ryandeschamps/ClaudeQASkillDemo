TEST SCRIPT: TS-072
Test Scenario: Enter Shipment Tracking Information
User Story: As a Logged-in Admin, I want to add shipping carrier, tracking ID, and delivery details, so that I can enable customer order tracking
Priority: High
Related Requirements: FR-017

GIVEN the admin is logged into the admin panel
  AND an order exists that is ready to ship
  AND the admin is viewing the order details page

WHEN the admin enters shipment tracking information:
  - Shipping carrier name (e.g., FedEx, UPS, USPS)
  - Tracking number/ID
  - Shipment date
  - Expected delivery date
  - Additional notes (optional)
  AND saves the information

THEN the system should:
  - Save all tracking information
  - Update order status to "Shipped" (if not already)
  - Display success confirmation message
  - Send shipping notification email to customer
  - Make tracking information visible to customer

WHEN the customer receives the shipping notification email

THEN the email should contain:
  - Shipment confirmation message
  - Order number
  - Tracking number
  - Carrier name
  - Expected delivery date
  - Link to track shipment

WHEN the customer views order details

THEN tracking information should be displayed
  AND tracking number should be clickable/copyable
  AND link to carrier tracking page should work

WHEN required tracking fields are missing

THEN the system should display validation errors
  AND prevent saving until all required fields are completed

EXPECTED RESULT:
- Tracking information is saved successfully
- All required fields are validated
- Customer receives shipping notification
- Tracking number is accessible to customer
- Carrier tracking links work correctly
- Expected delivery date is communicated
- Customer can track shipment progress
- Information helps reduce customer inquiries
