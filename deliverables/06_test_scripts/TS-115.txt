TEST SCRIPT: TS-115
Test Scenario: View Customer Complaints
User Story: As a Logged-in Admin, I want to see all customer inquiries and complaints, so that I can address customer issues
Priority: Medium
Related Requirements: FR-026

GIVEN the admin is logged into the admin panel
  AND customers have submitted inquiries/complaints

WHEN the admin navigates to "Support" or "Customer Complaints" section

THEN the system should display a list of all complaints/inquiries showing:
  - Ticket/Complaint ID
  - Customer name
  - Customer email
  - Subject/Topic
  - Submission date
  - Status (New, In Progress, Resolved, Closed)
  - Priority (if applicable)
  - Actions (View, Reply, Close)

WHEN there are many complaints

THEN the system should provide:
  - Pagination controls
  - Search functionality (by customer, subject, ticket ID)
  - Filter options (by status, date, priority)
  - Sort options (by date, status, priority)

WHEN the admin clicks on a complaint to view details

THEN the system should display:
  - Complete complaint/inquiry details
  - Customer information
  - Related order information (if applicable)
  - Complaint description
  - Submission timestamp
  - Status history
  - Reply/response capability
  - Option to change status

WHEN the admin filters by "New" status

THEN only unaddressed complaints should display

WHEN the admin searches for a customer

THEN all complaints from that customer should be shown

WHEN viewing complaint details

THEN the admin should be able to:
  - Reply to the customer
  - Change complaint status
  - Link to related order (if applicable)
  - Add internal notes
  - Assign to another admin (optional)

EXPECTED RESULT:
- All customer complaints are accessible
- Complaint information is complete
- Search and filter functionality works
- Admin can easily manage and respond to complaints
- Status tracking helps organize work
- Related order information is linked
- Priority complaints are identifiable
- Interface facilitates efficient support
