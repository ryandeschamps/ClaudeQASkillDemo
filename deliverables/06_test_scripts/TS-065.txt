TEST SCRIPT: TS-065
Test Scenario: View All Orders
User Story: As a Logged-in Admin, I want to see a list of all customer orders, so that I can manage order fulfillment
Priority: Critical
Related Requirements: FR-017

GIVEN the admin is logged into the admin panel

WHEN the admin navigates to "Order Management" or "Orders" section

THEN the system should display a list of all orders showing:
  - Order number/ID
  - Order date
  - Customer name
  - Order total amount
  - Payment status
  - Order status (Open, Confirmed, In Process, Shipped, Delivered)
  - Actions (View Details, Update Status)

WHEN there are many orders

THEN the system should provide:
  - Pagination controls
  - Records per page selector
  - Search functionality (by order number, customer name)
  - Filter options (by status, date range, payment status)
  - Sort options (by date, amount, status)
  - Date range picker for filtering orders

WHEN the admin uses search function

THEN the order list should filter to show only matching orders

WHEN the admin applies status filter

THEN only orders with selected status should display

WHEN the admin clicks on an order row or "View Details"

THEN the system should navigate to order details page

EXPECTED RESULT:
- All orders are listed comprehensively
- Order information is accurate and up-to-date
- List is paginated for performance
- Search, filter, and sort functionality works correctly
- Quick actions are available for each order
- Page loads within acceptable time
- Most recent orders appear first (default sorting)
- Admin can efficiently manage order fulfillment
