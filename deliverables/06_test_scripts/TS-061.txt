TEST SCRIPT: TS-061
Test Scenario: View Customer Details
User Story: As a Logged-in Admin, I want to view comprehensive details of a specific customer, so that I can understand their account and purchase history
Priority: High
Related Requirements: FR-016

GIVEN the admin is logged into the admin panel
  AND the admin is viewing the customer list

WHEN the admin clicks on a specific customer to view details

THEN the system should display the customer details page with:
  - Personal Information:
    * Full name
    * Email address
    * Phone number
    * Registration date
    * Account status
    * Email verification status
  - Address Book:
    * All saved addresses
    * Default billing/shipping addresses
  - Order History:
    * List of all orders with order number, date, status, total
    * Link to view each order details
  - Activity Summary:
    * Total orders placed
    * Total amount spent
    * Last login date
    * Number of reviews posted
  - Wishlist items (optional)
  - Actions available:
    * Edit customer information
    * Activate/Deactivate account
    * Reset password
    * Send email to customer

WHEN the admin clicks on an order in the history

THEN the system should navigate to that order's details page

WHEN viewing customer details

THEN all information should be current and accurate

EXPECTED RESULT:
- Complete customer profile is displayed
- All customer data is accurate
- Order history is comprehensive
- Navigation to related records works
- Admin has full visibility into customer account
- Page layout is organized and clear
- Admin can take appropriate actions from this page
