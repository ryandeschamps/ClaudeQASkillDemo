TEST SCRIPT: TS-038
Test Scenario: View Order Details
User Story: As a Logged-in Buyer, I want to view detailed information about a specific order, so that I can see what I ordered and its status
Priority: High
Related Requirements: FR-012

GIVEN the user is logged in
  AND the user has at least one order in their history
  AND the user is viewing the order history page

WHEN the user clicks on a specific order to view details

THEN the system should display the order details page with:
  - Order number
  - Order date and time
  - Current order status
  - List of items ordered:
    * Product image
    * Product name
    * Size and color
    * Quantity
    * Price per unit
    * Subtotal
  - Billing address
  - Shipping address
  - Payment method used
  - Pricing breakdown:
    * Items subtotal
    * Shipping charges
    * Tax
    * Total amount paid
  - Shipment tracking information (if order has shipped)
  - Option to download invoice
  - Option to reorder same items

WHEN the order has been shipped

THEN tracking information should be visible:
  - Carrier name
  - Tracking number
  - Expected delivery date
  - Link to track shipment

EXPECTED RESULT:
- Order details are complete and accurate
- All information matches the original order
- Current status is displayed correctly
- Tracking information is shown when available
- Invoice download works properly
- Page layout is clear and organized
- User can easily find all order information
