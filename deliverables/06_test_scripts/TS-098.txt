TEST SCRIPT: TS-098
Test Scenario: Export Report as PDF
User Story: As a Logged-in Admin, I want to export reports in PDF format, so that I can share them with stakeholders
Priority: Medium
Related Requirements: FR-021

GIVEN the admin is logged into the admin panel
  AND a report has been generated (revenue, products, or other report)
  AND the admin is viewing the report results

WHEN the admin clicks "Export as PDF" or "Download PDF" button

THEN the system should:
  - Generate a PDF version of the report
  - Include all report data and summary
  - Format the PDF professionally with:
    * Report title
    * Date range or parameters
    * Company logo/branding
    * Data tables with proper formatting
    * Page numbers
    * Generation date
    * Summary/totals section
  - Trigger download of the PDF file

WHEN the PDF is generated

THEN the file should:
  - Have a meaningful filename (e.g., "Revenue_Report_Jan2025.pdf")
  - Be properly formatted and readable
  - Include all data from the report
  - Preserve table structures and formatting
  - Be sized appropriately (not excessively large)

WHEN the report has multiple pages of data

THEN the PDF should:
  - Span multiple pages as needed
  - Include page headers/footers
  - Maintain consistent formatting across pages
  - Include totals and summaries

WHEN the download completes

THEN the admin should be able to:
  - Open and view the PDF
  - Print the PDF
  - Share the PDF via email
  - Archive the PDF for records

EXPECTED RESULT:
- PDF export functionality works correctly
- Generated PDF is professional and well-formatted
- All report data is included accurately
- File is downloadable and usable
- Formatting is consistent and readable
- PDFs are suitable for sharing with stakeholders
- File size is reasonable
- Works for all report types
