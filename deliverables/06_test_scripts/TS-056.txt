TEST SCRIPT: TS-056
Test Scenario: View Admin Dashboard Statistics
User Story: As a Logged-in Admin, I want to view key business metrics on the dashboard, so that I can quickly assess business performance
Priority: High
Related Requirements: FR-015

GIVEN the admin is logged into the admin panel

WHEN the admin accesses the dashboard homepage

THEN the system should display key metrics including:
  - Total number of registered buyers (active/inactive)
  - Total number of products in catalog
  - Revenue today
  - Revenue this month
  - Revenue this year (optional)
  - Number of orders today
  - Number of pending orders
  - Number of shipped orders
  - Recent orders list (last 5-10)
  - Low stock alerts (if applicable)
  - Pending reviews count
  - New customer complaints/inquiries

WHEN the dashboard loads

THEN all statistics should be:
  - Accurate and up-to-date
  - Calculated from real-time or recent data
  - Clearly labeled and organized
  - Presented in easy-to-read format (cards, charts, tables)

WHEN the admin clicks on a metric

THEN the system should navigate to detailed view of that metric
  (e.g., clicking "Pending Orders" shows order list filtered by pending status)

EXPECTED RESULT:
- Dashboard loads within acceptable time
- All metrics are accurate
- Data is current (not outdated)
- Visual presentation is clear and professional
- Quick links to detailed views work
- Dashboard provides comprehensive business overview
- Metrics update when underlying data changes
- Charts and graphs display correctly (if included)
