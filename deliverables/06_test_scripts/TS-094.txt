TEST SCRIPT: TS-094
Test Scenario: View Approved Reviews
User Story: As a Logged-in Admin, I want to see all approved reviews, so that I can monitor published customer feedback
Priority: Low
Related Requirements: FR-020

GIVEN the admin is logged into the admin panel
  AND reviews have been approved and published

WHEN the admin navigates to "Review Management" section
  AND filters by "Approved" status

THEN the system should display a list of approved reviews showing:
  - Product name and image
  - Customer name
  - Star rating
  - Review title and text
  - Approval date
  - Admin who approved
  - Number of helpful votes (if feature exists)
  - Actions (View, Unpublish/Reject)

WHEN there are many approved reviews

THEN the system should provide:
  - Pagination controls
  - Sort options (by date, rating, product, helpful votes)
  - Search by product or customer
  - Filter by rating (1-5 stars)
  - Filter by product category

WHEN the admin clicks on a review

THEN the system should display full review details
  AND show customer engagement metrics (if available)

WHEN viewing approved reviews

THEN the admin can:
  - Monitor customer feedback trends
  - Identify highly rated/poorly rated products
  - Spot potentially problematic reviews that were approved
  - Take action if review needs to be unpublished

WHEN the admin wants to unpublish an approved review

THEN the system should:
  - Allow changing status back to rejected
  - Require confirmation
  - Remove from public view
  - Log the action

EXPECTED RESULT:
- All approved reviews are accessible
- Review information is complete
- Admin can monitor published feedback
- Sort and filter options facilitate analysis
- Reviews can be unpublished if needed
- Approval audit trail is maintained
- Interface helps admin maintain review quality
- Customer feedback trends are visible
