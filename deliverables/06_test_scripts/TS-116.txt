TEST SCRIPT: TS-116
Test Scenario: Receive Email for New Complaint
User Story: As a Logged-in Admin, I want to receive email notifications when customers submit complaints, so that I can respond promptly
Priority: Medium
Related Requirements: FR-026

GIVEN the admin email notification is configured
  AND an admin email address is set in system settings

WHEN a customer submits a complaint or inquiry through:
  - Contact Us form
  - Support form
  - Email to support address

THEN the system should:
  - Create a support ticket/complaint record
  - Send email notification to admin email address
  - Include complaint details in the email

WHEN the notification email is sent

THEN it should contain:
  - Subject indicating new complaint/inquiry
  - Ticket/Complaint ID
  - Customer name and email
  - Subject/Topic of complaint
  - Complaint message/description
  - Submission timestamp
  - Link to view complaint in admin panel
  - Related order number (if applicable)

WHEN multiple admins should be notified

THEN the system should:
  - Send to all configured admin email addresses
  - Support email distribution lists
  - Allow configuring notification recipients

WHEN the email notification is received

THEN admins can:
  - Quickly review complaint details
  - Click link to access admin panel
  - Respond promptly to customer issue

WHEN email sending fails

THEN the system should:
  - Retry sending
  - Log the failure
  - Ensure complaint is still recorded in system

WHEN high priority or urgent complaints are submitted

THEN the notification email should:
  - Indicate priority level
  - Use subject line tags like [URGENT]
  - Draw admin attention

EXPECTED RESULT:
- Email notifications are sent for new complaints
- Notifications are received within minutes
- Email content is complete and informative
- Link to admin panel works correctly
- All configured admins receive notifications
- Priority complaints are highlighted
- Notification system is reliable
- Admins can respond quickly to customer issues
