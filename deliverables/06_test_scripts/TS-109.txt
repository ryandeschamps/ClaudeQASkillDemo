TEST SCRIPT: TS-109
Test Scenario: Edit Contact Us Page
User Story: As a Logged-in Admin, I want to update the Contact Us page content, so that I can provide accurate contact information
Priority: Medium
Related Requirements: FR-024

GIVEN the admin is logged into the admin panel
  AND a Contact Us page exists

WHEN the admin navigates to "Content Management" or "CMS" section
  AND selects "Contact Us" page
  AND clicks "Edit"

THEN the system should display a content editor with fields for:
  - Company address
  - Phone number(s)
  - Email address(es)
  - Business hours
  - Map embed code (if applicable)
  - Additional content (rich text editor)
  - Social media links

WHEN the admin modifies contact information:
  - Updates phone numbers
  - Changes email addresses
  - Updates physical address
  - Modifies business hours
  - Updates map location
  AND saves the changes

THEN the system should:
  - Validate email format
  - Validate phone number format
  - Update the Contact Us page
  - Display success confirmation message
  - Reflect changes on customer-facing site

WHEN the admin includes a map

THEN the system should:
  - Accept Google Maps or similar embed code
  - Display map correctly on the page
  - Ensure map is responsive

WHEN the admin updates social media links

THEN the system should:
  - Validate URL format
  - Update social media icons/links
  - Ensure links open correctly

WHEN the changes are saved

THEN the customer-facing Contact Us page should:
  - Display updated contact information
  - Show correct business hours
  - Display map if included
  - Show working social media links

EXPECTED RESULT:
- Contact information can be updated successfully
- All contact fields are editable
- Validation ensures correct formats
- Map integration works correctly
- Social media links are functional
- Changes are reflected immediately on customer site
- Page displays correctly with updated information
- Contact form continues to work (if present)
