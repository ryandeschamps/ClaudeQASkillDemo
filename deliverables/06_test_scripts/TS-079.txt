TEST SCRIPT: TS-079
Test Scenario: View Product Catalog
User Story: As a Logged-in Admin, I want to see all products in the catalog, so that I can manage the product inventory
Priority: High
Related Requirements: FR-019

GIVEN the admin is logged into the admin panel

WHEN the admin navigates to "Product Management" or "Products" section

THEN the system should display a list of all products showing:
  - Product ID
  - Product image (thumbnail)
  - Product name
  - Category/Sub-category
  - Price
  - Stock status
  - Status (Active/Inactive)
  - Created date
  - Actions (View, Edit, Activate/Deactivate, Delete)

WHEN there are many products

THEN the system should provide:
  - Pagination controls
  - Records per page selector
  - Search functionality (by name, SKU, ID)
  - Filter options (by category, status, stock status)
  - Sort options (by name, price, date, status)

WHEN the admin uses search function

THEN the product list should filter to show only matching products

WHEN the admin applies category filter

THEN only products in selected category should display

WHEN the admin clicks on a product row or "View" action

THEN the system should navigate to product details page

WHEN the admin clicks "Edit"

THEN the system should navigate to product edit page

EXPECTED RESULT:
- All products are listed comprehensively
- Product information is accurate and current
- List is paginated for performance
- Thumbnail images display correctly
- Search and filter functionality works
- Quick actions are available for each product
- Page loads within acceptable time
- Admin can efficiently browse and manage products
- Stock status is clearly indicated
