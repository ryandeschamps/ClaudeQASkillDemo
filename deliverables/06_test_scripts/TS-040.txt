TEST SCRIPT: TS-040
Test Scenario: Receive Order Status Update Emails
User Story: As a Buyer with active orders, I want to receive email notifications when my order status changes, so that I can stay informed about my order progress
Priority: High
Related Requirements: FR-008

GIVEN the user has placed an order
  AND the order is being processed

WHEN the admin updates the order status to "Confirmed"

THEN the system should send an email notification to the buyer
  AND the email should contain:
    - Order confirmation message
    - Order number
    - Estimated processing time

WHEN the admin updates the order status to "In Process"

THEN the system should send an email notification
  AND the email should inform that order is being prepared

WHEN the admin updates the order status to "Shipped"

THEN the system should send an email notification
  AND the email should contain:
    - Shipment confirmation
    - Tracking number
    - Carrier name
    - Expected delivery date
    - Link to track shipment

WHEN the order status changes to "Delivered"

THEN the system should send an email notification
  AND the email should:
    - Confirm successful delivery
    - Request feedback/review
    - Provide customer support information

EXPECTED RESULT:
- Email notifications are sent for each status change
- Emails are received within minutes of status update
- Email content is relevant to each status
- Tracking information is included when shipped
- All order details are accurate in emails
- Email format is professional and clear
- Unsubscribe option is available (if applicable)
