TEST SCRIPT: TS-052
Test Scenario: Contact Customer Support
User Story: As a Logged-in Buyer, I want to send a message to customer support, so that I can get help with issues or questions
Priority: Medium
Related Requirements: FR-013

GIVEN the user is logged in
  AND the user is on any page of the website

WHEN the user navigates to "Contact Support" or "Help" page
  AND fills in the support form with:
    - Subject/Topic
    - Message/Description
    - Order number (if applicable)
    - Priority level (optional)
  AND submits the form

THEN the system should:
  - Validate all required fields
  - Auto-populate user's name and email from account
  - Save the support ticket
  - Send the inquiry to customer support team
  - Display confirmation message with ticket number
  - Send confirmation email to user

WHEN the user includes an order number

THEN the system should:
  - Validate the order number belongs to the user
  - Link the support ticket to that order
  - Provide order context to support team

WHEN required fields are missing

THEN the system should display validation errors
  AND prevent submission

WHEN the support ticket is created

THEN the user should be able to:
  - View ticket status in account dashboard
  - Receive updates when support responds

EXPECTED RESULT:
- Support form is easily accessible
- Form pre-fills user information
- Submission is successful
- Ticket number is generated
- User receives confirmation
- Support team receives the inquiry with full context
- User can track ticket status
- Response notifications work correctly
