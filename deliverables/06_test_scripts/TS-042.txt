TEST SCRIPT: TS-042
Test Scenario: View Account Dashboard
User Story: As a Logged-in Buyer, I want to access my account dashboard, so that I can manage all aspects of my account from one place
Priority: High
Related Requirements: FR-010

GIVEN the user is logged in

WHEN the user clicks on "My Account" or account icon
  AND navigates to the account dashboard

THEN the system should display the account dashboard with:
  - Welcome message with user's name
  - Quick links/sections for:
    * Order history
    * Wishlist
    * Profile information
    * Address book
    * Password change
    * Payment methods (if saved)
  - Summary information:
    * Number of orders
    * Number of wishlist items
    * Recent order status (if applicable)
  - Logout option

WHEN the user clicks on any section link

THEN the system should navigate to that specific section
  AND display relevant information

WHEN the user has no orders yet

THEN the dashboard should display "No orders yet" message
  AND provide link to browse products

EXPECTED RESULT:
- Dashboard is accessible and loads quickly
- All sections are clearly organized
- Summary information is accurate
- Navigation to different sections works smoothly
- Dashboard provides good overview of account status
- User interface is intuitive and user-friendly
- Quick access to frequently used features
