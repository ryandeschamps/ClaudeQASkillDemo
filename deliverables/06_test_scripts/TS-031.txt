TEST SCRIPT: TS-031
Test Scenario: View Order Summary Before Payment
User Story: As a Buyer in Checkout, I want to review complete order summary including items, prices, shipping, and tax, so that I can verify everything is correct before payment
Priority: Critical
Related Requirements: FR-008

GIVEN the user is logged in
  AND the user has completed address entry steps
  AND the user is on the order review/summary step before payment

WHEN the order summary page loads

THEN the system should display:
  - All cart items with:
    * Product image
    * Product name
    * Size and color
    * Quantity
    * Price per unit
    * Subtotal per item
  - Billing address
  - Shipping address
  - Pricing breakdown:
    * Items subtotal
    * Shipping charges
    * Tax amount
    * Order total
  - "Edit" options for each section
  - "Proceed to Payment" button

WHEN the user clicks "Edit" for any section

THEN the system should navigate back to that step
  AND allow modifications
  AND return to order summary after changes

WHEN all information is correct and user clicks "Proceed to Payment"

THEN the system should navigate to payment page
  AND preserve all order details

EXPECTED RESULT:
- Order summary is complete and accurate
- All calculations are correct
- User can review all order details in one place
- Edit functionality works for all sections
- Visual layout is clear and organized
- User feels confident before proceeding to payment
