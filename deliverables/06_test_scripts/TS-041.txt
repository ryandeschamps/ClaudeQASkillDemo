TEST SCRIPT: TS-041
Test Scenario: Reorder Previous Items
User Story: As a Logged-in Buyer, I want to reorder items from a previous order, so that I can quickly purchase products I've bought before
Priority: Medium
Related Requirements: FR-012

GIVEN the user is logged in
  AND the user has at least one completed order in their history
  AND the user is viewing order details or order history

WHEN the user clicks the "Reorder" or "Buy Again" button for a previous order

THEN the system should:
  - Retrieve all items from the selected order
  - Check current availability of each product
  - Add available items to the shopping cart
  - Display confirmation message showing which items were added

WHEN some items from the order are no longer available

THEN the system should:
  - Add only the available items to cart
  - Display a message listing unavailable items
  - Provide option to browse alternatives

WHEN product prices have changed since original order

THEN the system should:
  - Add items at current prices
  - Optionally inform user of price changes

WHEN the user proceeds to checkout after reordering

THEN the system should follow normal checkout flow
  AND allow updating quantities or selections

EXPECTED RESULT:
- Reorder functionality works correctly
- Available items are added to cart successfully
- Unavailable items are handled gracefully
- User is informed of any changes (availability, price)
- Cart can be modified before checkout
- Feature saves time for repeat customers
- User experience is smooth and intuitive
