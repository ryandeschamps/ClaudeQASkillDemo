TEST SCRIPT: TS-084
Test Scenario: Activate Product
User Story: As a Logged-in Admin, I want to activate a product, so that I can make it visible and purchasable on the website
Priority: High
Related Requirements: FR-019

GIVEN the admin is logged into the admin panel
  AND a product exists with "Inactive" status
  AND the admin is viewing the product list or product details

WHEN the admin clicks "Activate" button for the inactive product
  AND confirms the action (if confirmation required)

THEN the system should:
  - Change product status to "Active"
  - Update the product record
  - Display success confirmation message
  - Make product visible on customer-facing pages
  - Make product searchable and browsable
  - Allow product to be added to cart

WHEN the product is activated

THEN the product should appear in:
  - Category/sub-category listings
  - Search results
  - Product catalog
  - Filtered results (if matching filter criteria)

WHEN the product has no images or incomplete information

THEN the system should either:
  - Warn admin before activating
  - Still allow activation (admin decision)
  - Display validation warnings for missing recommended fields

WHEN viewing the product list

THEN the activated product should show "Active" status
  AND be included in active products count

WHEN a customer browses the category

THEN the activated product should be immediately visible

EXPECTED RESULT:
- Product activation is successful
- Product becomes immediately visible to customers
- Status change is reflected throughout system
- Customers can search and find the product
- Product can be added to cart and purchased
- Admin can easily activate products
- Activation is reversible by deactivating
- Optional warnings for incomplete product data
