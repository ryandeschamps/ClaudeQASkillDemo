TEST SCRIPT: TS-091
Test Scenario: View Pending Reviews
User Story: As a Logged-in Admin, I want to see all reviews awaiting moderation, so that I can approve or reject them
Priority: Medium
Related Requirements: FR-020

GIVEN the admin is logged into the admin panel
  AND customer reviews have been submitted
  AND some reviews are in "Pending" status awaiting moderation

WHEN the admin navigates to "Review Management" or "Reviews" section
  AND filters by "Pending" status

THEN the system should display a list of pending reviews showing:
  - Product name and image
  - Customer name
  - Star rating given
  - Review title
  - Review text (excerpt or full)
  - Submission date
  - Actions (Approve, Reject, View Details)

WHEN there are many pending reviews

THEN the system should provide:
  - Pagination controls
  - Sort options (by date, rating, product)
  - Search by product name or customer
  - Bulk actions (approve/reject multiple)

WHEN the admin clicks on a review to view details

THEN the system should display:
  - Complete review content
  - Customer information
  - Product details
  - Submission timestamp
  - Customer's purchase verification status
  - Any attached photos (if applicable)

WHEN no pending reviews exist

THEN the system should display "No pending reviews" message

WHEN new reviews are submitted

THEN the pending review count should update
  AND new reviews should appear in the list

EXPECTED RESULT:
- All pending reviews are listed
- Review information is complete
- Admin can easily review content before approval
- Purchase verification status is shown
- Sort and filter options work correctly
- Bulk actions available for efficiency
- Review count is accurate
- Interface facilitates quick moderation decisions
