TEST SCRIPT: TS-108
Test Scenario: Edit About Us Page
User Story: As a Logged-in Admin, I want to update the About Us page content, so that I can keep company information current
Priority: Medium
Related Requirements: FR-024

GIVEN the admin is logged into the admin panel
  AND an About Us page exists

WHEN the admin navigates to "Content Management" or "CMS" section
  AND selects "About Us" page
  AND clicks "Edit"

THEN the system should display a content editor with:
  - Rich text editor (WYSIWYG)
  - Current page content
  - Formatting tools (bold, italic, lists, headings)
  - Image upload capability
  - HTML editing option (advanced)
  - Preview option

WHEN the admin modifies the content:
  - Updates company history
  - Changes team information
  - Adds/removes images
  - Updates contact information
  AND clicks "Save" or "Publish"

THEN the system should:
  - Validate content (if applicable)
  - Save the updated content
  - Display success confirmation message
  - Update the About Us page on customer-facing site
  - Preserve content formatting

WHEN the admin clicks "Preview"

THEN the system should display how the page will look on the website
  WITHOUT publishing changes

WHEN the content includes images

THEN the system should:
  - Allow uploading new images
  - Support image positioning and sizing
  - Store images properly
  - Display images correctly on published page

WHEN the admin saves changes

THEN the customer-facing About Us page should:
  - Immediately reflect the updates
  - Maintain proper formatting
  - Display correctly on all devices

EXPECTED RESULT:
- About Us page content can be edited successfully
- Rich text editor provides necessary formatting tools
- Images can be uploaded and positioned
- Preview functionality works correctly
- Changes are published to customer-facing site
- Formatting is preserved
- Content updates are immediate
- Page displays correctly after update
