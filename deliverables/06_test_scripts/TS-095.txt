TEST SCRIPT: TS-095
Test Scenario: View Rejected Reviews
User Story: As a Logged-in Admin, I want to see all rejected reviews, so that I can maintain a record of moderation decisions
Priority: Low
Related Requirements: FR-020

GIVEN the admin is logged into the admin panel
  AND reviews have been rejected

WHEN the admin navigates to "Review Management" section
  AND filters by "Rejected" status

THEN the system should display a list of rejected reviews showing:
  - Product name and image
  - Customer name
  - Star rating
  - Review title and text
  - Submission date
  - Rejection date
  - Admin who rejected
  - Rejection reason (if recorded)
  - Actions (View Details, Approve)

WHEN viewing rejected reviews

THEN the system should show:
  - Why review was rejected
  - Who made the rejection decision
  - When it was rejected
  - Option to reverse decision

WHEN there are many rejected reviews

THEN the system should provide:
  - Pagination controls
  - Sort options (by rejection date, product, admin)
  - Search functionality
  - Filter by rejection reason

WHEN the admin wants to approve a previously rejected review

THEN the system should:
  - Allow changing status to "Approved"
  - Require confirmation
  - Move to approved reviews list
  - Publish on product page
  - Log the status change

WHEN viewing rejection history

THEN the admin can:
  - Track moderation patterns
  - Review moderation decisions for quality control
  - Train new moderators using examples
  - Ensure consistent moderation standards

EXPECTED RESULT:
- All rejected reviews are accessible
- Rejection reasons are documented
- Rejection audit trail is complete
- Admin can review moderation decisions
- Rejected reviews can be approved if decision was wrong
- Sort and filter facilitate review management
- History helps maintain moderation consistency
- Interface supports quality control processes
