TEST SCRIPT: TS-060
Test Scenario: View Customer List
User Story: As a Logged-in Admin, I want to see a list of all registered customers, so that I can manage customer accounts
Priority: High
Related Requirements: FR-016

GIVEN the admin is logged into the admin panel

WHEN the admin navigates to "Customer Management" or "Customers" section

THEN the system should display a list of all registered buyers showing:
  - Customer ID or number
  - Full name
  - Email address
  - Phone number
  - Registration date
  - Account status (Active/Inactive)
  - Total orders count
  - Actions (View, Edit, Activate/Deactivate)

WHEN there are many customers

THEN the system should provide:
  - Pagination controls
  - Number of records per page selector
  - Search functionality (by name, email, phone)
  - Filter options (by status, registration date)
  - Sort options (by name, date, order count)

WHEN the admin uses the search function

THEN the list should filter to show only matching customers

WHEN the admin clicks on a customer row or "View" action

THEN the system should navigate to customer details page

WHEN the admin applies filters

THEN the list should update to show only customers matching filter criteria

EXPECTED RESULT:
- All registered customers are listed
- Customer information is accurate and complete
- List is paginated for performance
- Search and filter functionality works correctly
- Sorting options function properly
- Quick actions are available for each customer
- Page loads within acceptable time even with many customers
- Admin can easily find and manage customers
