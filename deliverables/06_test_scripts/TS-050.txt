TEST SCRIPT: TS-050
Test Scenario: View My Posted Reviews
User Story: As a Logged-in Buyer, I want to see all reviews I have posted, so that I can track my feedback history
Priority: Low
Related Requirements: FR-010, FR-011

GIVEN the user is logged in
  AND the user has posted reviews for products

WHEN the user navigates to "My Reviews" section in account dashboard

THEN the system should display a list of all posted reviews showing:
  - Product name and image
  - Star rating given
  - Review title and text
  - Date posted
  - Review status (Pending, Approved, Rejected)
  - Option to edit or delete (if allowed)

WHEN the user clicks on a product in the review list

THEN the system should navigate to that product's details page

WHEN the user has reviews in different statuses

THEN the system should:
  - Clearly indicate status of each review
  - Show approved reviews differently from pending/rejected
  - Optionally provide filters to view by status

WHEN the user has not posted any reviews

THEN the system should display "No reviews yet" message
  AND provide link to browse products to review

WHEN the user clicks "Edit" on a pending review

THEN the system should allow editing the review
  AND resubmit for moderation

EXPECTED RESULT:
- All user's reviews are displayed correctly
- Review status is clearly indicated
- User can track moderation status
- Navigation to products works
- Edit functionality is available for pending reviews
- List is organized chronologically (newest first)
- Page layout is clear and informative
