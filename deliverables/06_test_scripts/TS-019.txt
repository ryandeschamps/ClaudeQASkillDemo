TEST SCRIPT: TS-019
Test Scenario: View Shopping Cart
User Story: As a Logged-in Buyer, I want to view all items in my shopping cart, so that I can review my selections before checkout
Priority: Critical
Related Requirements: FR-007

GIVEN the user is logged in
  AND the user has items in their shopping cart

WHEN the user clicks on the shopping cart icon or navigates to cart page

THEN the system should display the shopping cart page with:
  - List of all cart items showing:
    * Product image
    * Product name
    * Selected size and color
    * Price per unit
    * Quantity
    * Subtotal per item
  - Cart summary showing:
    * Subtotal of all items
    * Estimated shipping charges (if calculable)
    * Estimated tax
    * Total amount
  - Options to update quantity or remove items
  - "Proceed to Checkout" button

WHEN the cart is empty

THEN the system should display "Your cart is empty" message
  AND provide a link to continue shopping

EXPECTED RESULT:
- All cart items are displayed correctly
- Product information matches selections
- Prices and calculations are accurate
- Cart summary totals are correct
- Update and remove options are functional
- Empty cart is handled gracefully
- Page loads within acceptable time
