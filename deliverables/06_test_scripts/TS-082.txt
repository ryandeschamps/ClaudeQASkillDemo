TEST SCRIPT: TS-082
Test Scenario: Edit Product Details
User Story: As a Logged-in Admin, I want to modify existing product information, so that I can update prices, descriptions, or other attributes
Priority: High
Related Requirements: FR-019

GIVEN the admin is logged into the admin panel
  AND a product exists in the catalog
  AND the admin is viewing the product list

WHEN the admin clicks "Edit" for a specific product
  AND modifies product information:
    - Product name
    - Description
    - Price
    - Category/Sub-category
    - Available sizes
    - Available colors
    - Stock quantity
    - Product specifications
  AND saves the changes

THEN the system should:
  - Validate all required fields
  - Validate price is a positive number
  - Validate stock quantity is non-negative
  - Update the product record
  - Display success confirmation message
  - Reflect changes on product detail page immediately

WHEN the admin changes the price

THEN the new price should be reflected:
  - On product listing pages
  - On product detail page
  - In shopping carts containing this product (or prompt user to refresh)

WHEN required fields are left empty

THEN the system should display validation errors
  AND prevent saving until corrections are made

WHEN the admin updates product images

THEN new images should replace or be added to existing images
  AND display correctly on customer-facing pages

WHEN the admin changes category assignment

THEN the product should appear in the new category
  AND be removed from old category navigation

EXPECTED RESULT:
- Product information updates successfully
- All validations work correctly
- Changes are reflected throughout the system
- Customer-facing pages show updated information
- Price changes take effect immediately
- Category changes update navigation
- Stock quantity updates are accurate
- No data is lost during editing
