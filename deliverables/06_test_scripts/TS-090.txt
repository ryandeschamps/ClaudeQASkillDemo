TEST SCRIPT: TS-090
Test Scenario: View Order Payment Status
User Story: As a Logged-in Admin, I want to see the payment status of each order, so that I can verify successful transactions
Priority: High
Related Requirements: FR-020

GIVEN the admin is logged into the admin panel
  AND orders exist in the system with various payment statuses

WHEN the admin views the order list

THEN each order should display payment status such as:
  - Pending
  - Paid
  - Failed
  - Refunded
  - Partially Refunded

WHEN the admin clicks on an order to view details

THEN the order details page should display:
  - Payment status
  - Payment method used (Credit Card, Debit Card, Net Banking)
  - Transaction ID from payment gateway
  - Payment date and time
  - Amount paid
  - Payment gateway response
  - Last 4 digits of card (if card payment)
  - Bank name (if net banking)

WHEN payment has failed

THEN the system should display:
  - Failure reason
  - Error message from gateway
  - Option to retry payment (if applicable)

WHEN viewing payment status in order list

THEN the admin should be able to:
  - Filter orders by payment status
  - Sort by payment date
  - Search by transaction ID
  - Identify unpaid orders quickly

WHEN payment is successful

THEN the status should show "Paid" or "Success"
  AND transaction ID should be recorded

EXPECTED RESULT:
- Payment status is clearly visible for all orders
- Transaction details are comprehensive
- Failed payments show reason for failure
- Payment method is identified
- Transaction IDs are recorded and displayed
- Filter and search by payment status works
- Admin can easily verify payment completion
- Sensitive payment data is masked appropriately
- Payment gateway responses are logged
