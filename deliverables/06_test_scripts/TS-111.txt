TEST SCRIPT: TS-111
Test Scenario: Edit Terms and Conditions Page
User Story: As a Logged-in Admin, I want to update the Terms and Conditions content, so that I can maintain legal compliance
Priority: High
Related Requirements: FR-024

GIVEN the admin is logged into the admin panel
  AND a Terms and Conditions page exists

WHEN the admin navigates to "Content Management" or "CMS" section
  AND selects "Terms and Conditions" page
  AND clicks "Edit"

THEN the system should display a content editor with:
  - Rich text editor (WYSIWYG)
  - Current terms and conditions content
  - Formatting tools for legal document structure
  - Numbered sections/clauses capability
  - Last updated date field
  - Version tracking (optional)

WHEN the admin modifies the terms and conditions:
  - Updates service terms
  - Modifies payment terms
  - Changes refund/return policies
  - Updates liability clauses
  - Modifies dispute resolution terms
  AND saves the changes

THEN the system should:
  - Update the Terms and Conditions page
  - Update "Last Modified" date automatically
  - Display success confirmation message
  - Publish changes to customer-facing site
  - Optionally archive previous version

WHEN the admin organizes content

THEN the editor should support:
  - Numbered sections and subsections
  - Bullet points and lists
  - Headings and subheadings
  - Bold/italic for emphasis
  - Proper legal document formatting

WHEN the admin saves changes

THEN the system should:
  - Timestamp the update
  - Maintain version history (if implemented)
  - Display new "Last Updated" date on customer page
  - Notify users of changes (optional, based on policy)

WHEN customers view the Terms and Conditions

THEN they should see:
  - Updated terms content
  - Last updated date
  - Properly formatted sections
  - Clear and readable layout

EXPECTED RESULT:
- Terms and Conditions can be updated successfully
- Legal document formatting is supported
- Last updated date is tracked automatically
- Changes are published immediately
- Customer-facing page shows updated terms
- Legal compliance can be maintained
- Proper formatting for legal content
- Version tracking works (if implemented)
