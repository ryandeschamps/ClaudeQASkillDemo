TEST SCRIPT: TS-016
Test Scenario: Reset Forgotten Password
User Story: As a Buyer, I want to reset my password if I forget it, so that I can regain access to my account
Priority: High
Related Requirements: FR-001

GIVEN the user is on the login page
  AND the user has a registered account
  AND the user has forgotten their password

WHEN the user clicks on "Forgot Password" link
  AND enters their registered email address
  AND submits the form

THEN the system should:
  - Validate the email address exists in the system
  - Send a password reset link to the email address
  - Display a confirmation message

WHEN the user clicks the reset link in the email
  AND enters a new password (meeting password requirements)
  AND confirms the new password

THEN the system should:
  - Validate the reset token
  - Validate password meets requirements
  - Update the password in the database
  - Display success message
  - Redirect to login page

WHEN the reset link is expired or invalid

THEN the system should display an error message
  AND provide option to request a new reset link

EXPECTED RESULT:
- Password reset email is sent successfully
- Reset link is valid and secure (token-based)
- New password meets complexity requirements
- Password is updated successfully
- User can login with new password
- Old password no longer works
- Reset tokens expire after configured time
- Security: tokens are one-time use only
