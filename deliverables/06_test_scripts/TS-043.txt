TEST SCRIPT: TS-043
Test Scenario: Update Profile Information
User Story: As a Logged-in Buyer, I want to update my email address and phone number, so that I can keep my contact information current
Priority: High
Related Requirements: FR-010

GIVEN the user is logged in
  AND the user is on the account dashboard or profile page

WHEN the user navigates to "Profile Information" or "Edit Profile"
  AND modifies their email address and/or phone number
  AND clicks "Save Changes"

THEN the system should:
  - Validate the new email format
  - Validate the new phone number format
  - Check if new email is already used by another account
  - Update the profile information
  - Display success confirmation message

WHEN the new email address is already registered

THEN the system should display error message
  AND not update the email
  AND keep the current email address

WHEN email is changed

THEN the system should:
  - Send verification email to new address
  - Require email verification before update is complete
  - Optionally send notification to old email address

WHEN required fields are left empty

THEN the system should display validation errors
  AND prevent saving until all required fields are completed

EXPECTED RESULT:
- Profile information updates successfully
- Email uniqueness is enforced
- Email verification is required for email changes
- Phone number format is validated
- Success/error messages are clear
- Updated information is saved persistently
- User receives confirmation of changes
