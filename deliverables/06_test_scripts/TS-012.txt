TEST SCRIPT: TS-012
Test Scenario: Verify Email Address
User Story: As a Registered Buyer, I want to verify my email address through a verification link, so that I can activate my account and start shopping
Priority: Critical
Related Requirements: FR-002

GIVEN a new user has completed registration
  AND a verification email has been sent to the registered email address
  AND the account is in unverified status

WHEN the user clicks on the verification link in the email

THEN the system should validate the verification token
  AND mark the account as verified
  AND display a success message
  AND redirect the user to the login page or auto-login the user

WHEN the verification link is expired (after configured time period)

THEN the system should display an "expired link" message
  AND provide an option to resend the verification email

WHEN the verification link is invalid or already used

THEN the system should display an appropriate error message
  AND provide instructions for next steps

EXPECTED RESULT:
- Verification email is received within acceptable time
- Verification link works correctly
- Account status updates to verified upon successful verification
- Expired link handling works properly
- User can resend verification email if needed
- Security: verification tokens are one-time use
- User experience is smooth and clear
