TEST SCRIPT: TS-027
Test Scenario: Enter Billing Address
User Story: As a Buyer in Checkout, I want to provide my billing address, so that I can complete the payment process
Priority: Critical
Related Requirements: FR-008

GIVEN the user is logged in
  AND the user has items in their cart
  AND the user is on the checkout page - billing address step

WHEN the user enters billing address information:
  - Full name
  - Address line 1
  - Address line 2 (optional)
  - City
  - State/Province
  - ZIP/Postal code
  - Country
  - Phone number
  AND submits the form

THEN the system should:
  - Validate all required fields are completed
  - Validate format of phone number and postal code
  - Save the billing address
  - Proceed to next checkout step (shipping address or payment)

WHEN the user selects a saved billing address from address book

THEN the system should populate all fields automatically
  AND allow the user to proceed to next step

WHEN required fields are missing or invalid

THEN the system should display appropriate validation error messages
  AND prevent proceeding to next step

EXPECTED RESULT:
- All billing address fields are captured correctly
- Field validation works properly
- Saved addresses can be selected and used
- Option to save new address for future use is available
- Form is user-friendly and clear
- Validation errors are specific and helpful
