TEST SCRIPT: TS-096
Test Scenario: View Products Uploaded Report
User Story: As a Logged-in Admin, I want to generate a report of products uploaded by date range, so that I can analyze catalog growth
Priority: Medium
Related Requirements: FR-021

GIVEN the admin is logged into the admin panel
  AND products have been added to the catalog over time

WHEN the admin navigates to "Reports" section
  AND selects "Products Uploaded Report"
  AND specifies a date range (start date and end date)
  AND generates the report

THEN the system should display a report showing:
  - List of products added in the date range
  - Product name
  - Product ID/SKU
  - Category
  - Upload date
  - Admin who added the product
  - Product status (Active/Inactive)
  - Initial price
  - Total count of products uploaded in period

WHEN the date range is specified

THEN the system should:
  - Include only products added within the date range
  - Display results sorted by upload date (newest first or oldest first)
  - Show summary statistics (total products, by category, by status)

WHEN viewing the report

THEN the admin should be able to:
  - Export report as PDF
  - Export report as Excel
  - Filter by category
  - Filter by admin user
  - Sort by different columns

WHEN no products were uploaded in the date range

THEN the system should display "No products found for selected period"

WHEN a large number of products exist

THEN the report should:
  - Paginate results
  - Load within acceptable time
  - Provide summary totals

EXPECTED RESULT:
- Report generates successfully
- Data is accurate for selected date range
- All product information is included
- Export functionality works (PDF, Excel)
- Summary statistics are helpful
- Report helps analyze catalog growth
- Filters and sorting options work correctly
- Performance is acceptable for large datasets
