TEST SCRIPT: TS-059
Test Scenario: View Revenue Statistics
User Story: As a Logged-in Admin, I want to view revenue for today and this month, so that I can track sales performance
Priority: High
Related Requirements: FR-015

GIVEN the admin is logged into the admin panel
  AND the admin is viewing the dashboard
  AND there are completed orders in the system

WHEN the dashboard displays revenue metrics

THEN the system should show:
  - Revenue today (total sales for current day)
  - Revenue this month (total sales for current month)
  - Optional: Revenue this year
  - Optional: Comparison to previous period
  - Optional: Revenue trends/charts

WHEN calculating revenue

THEN the system should:
  - Include only completed/delivered orders
  - Exclude cancelled or pending orders
  - Sum the total amounts paid
  - Display in correct currency format

WHEN the admin clicks on revenue metrics

THEN the system should navigate to detailed revenue reports
  OR display breakdown by:
    - Orders contributing to revenue
    - Revenue by category
    - Revenue by day/week

WHEN a new order is completed

THEN today's revenue should update to include the new order total

WHEN viewing at the start of a new month

THEN month's revenue should reset to 0
  AND start accumulating from first order of new month

EXPECTED RESULT:
- Revenue calculations are accurate
- Only successful orders are included
- Today's revenue updates in real-time
- Month's revenue accumulates correctly
- Currency formatting is correct
- Comparison metrics are helpful (if included)
- Charts/graphs display clearly (if included)
- Admin can track sales performance easily
