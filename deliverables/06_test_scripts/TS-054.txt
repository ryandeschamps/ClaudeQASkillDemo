TEST SCRIPT: TS-054
Test Scenario: Admin Password Reset
User Story: As an Admin User, I want to reset my password if forgotten, so that I can regain access to the admin panel
Priority: High
Related Requirements: FR-014

GIVEN an admin user exists in the system
  AND the admin user has forgotten their password
  AND the admin user is on the admin login page

WHEN the admin clicks "Forgot Password" link
  AND enters their admin username or email
  AND submits the request

THEN the system should:
  - Validate the username/email exists in admin accounts
  - Generate a secure password reset token
  - Send reset link to admin's registered email
  - Display confirmation message

WHEN the admin clicks the reset link in the email
  AND enters a new password meeting requirements
  AND confirms the new password

THEN the system should:
  - Validate the reset token is valid and not expired
  - Validate password meets security requirements
  - Update the admin password
  - Display success message
  - Redirect to admin login page

WHEN the reset link is expired or invalid

THEN the system should display error message
  AND provide option to request new reset link

WHEN the new password doesn't meet requirements

THEN the system should display specific requirements
  AND prevent password update

EXPECTED RESULT:
- Password reset email is sent to admin
- Reset link is secure and time-limited
- New password meets complexity requirements
- Password is updated successfully
- Admin can login with new password
- Old password no longer works
- Reset tokens are one-time use
- Security logging is maintained
