Test Script ID: TS-038
Test Scenario: Administrator Reject Documents
Priority: High
Preconditions:
- Benefits administrator role with document review authority
- Documents uploaded that do not meet requirements
- Rejection reason codes configured
- Employee notification process for rejected documents
- Resubmission workflow enabled

GIVEN: An employee uploaded an invalid or incomplete document that cannot be approved
WHEN: Benefits administrator reviews and rejects the document with specific reason
THEN: Document rejected, employee notified with reason, and can resubmit corrected document

Test Steps:
1. Benefits administrator logs into Oracle Fusion HCM
2. Navigate to "Document Approvals" queue
3. Filter to show "Pending Approval" documents
4. Select document requiring rejection: Birth certificate for Patricia Martinez
5. Click to view document details:
   - Employee: Patricia Martinez (P100006)
   - Document type: Birth certificate
   - Upload date: 2024-02-22
   - Life event: New hire enrollment
   - Status: Pending Approval
6. Click "View Document" to review uploaded file
7. Identify issues with document:
   - Document is illegible/blurry
   - Missing required information (date of birth cut off)
   - Wrong document uploaded (marriage certificate instead of birth certificate)
   - Document appears altered or fraudulent
8. Document fails verification checks
9. Click "Reject Document"
10. System displays rejection reason code options:
    - Document illegible
    - Incomplete information
    - Wrong document type
    - Suspected fraud
    - Does not match records
    - Other (specify)
11. Select reason: "Document illegible - please resubmit clearer copy"
12. Enter detailed rejection notes: "Image is blurry and child's date of birth is not readable"
13. Confirm rejection
14. System updates document status to "Rejected"
15. Action item remains open with status "Resubmission Required"
16. System sends notification to employee:
    - Subject: "Document Rejected - Action Required"
    - Body: Explains rejection reason and resubmission instructions
17. Employee receives email and portal notification with rejection details
18. Employee uploads corrected document (clearer image)
19. New submission enters administrator approval queue
20. Administrator reviews resubmission and approves if acceptable
21. Generate rejection activity report for quality monitoring

Expected Result:
- Administrator successfully rejects invalid document
- Rejection reason codes provide clear, specific feedback
- Detailed rejection notes captured for employee understanding
- Document status updates to "Rejected"
- Action item remains open requiring resubmission
- Employee receives timely notification of rejection
- Rejection reason clearly communicated to employee
- Instructions for resubmission provided
- Employee can upload corrected document
- Resubmission enters approval queue for review
- If enrollment suspended, remains suspended until valid document approved
- Audit trail captures rejection, reason, administrator ID, date
- Rejection trends reportable for quality improvement

Test Data Reference:
- Variant IDs: V006, V013, V049 (Document_Status: Rejected)
- Test Data: Row 7 (P100006, Patricia Martinez)
  - Document_Type: Birth Certificate
  - Document_Status: Rejected
  - Action_Item_Status: Suspended

Notes:
- Verify rejection reasons are clear and actionable for employees
- Confirm employee notification includes specific instructions for correction
- Test that rejected documents can be resubmitted
- Verify action items remain open until valid document approved
- Ensure enrollment remains suspended if coverage conditional on document
- Test multiple rejection cycles if needed
- Confirm audit trail for all rejection activity
- Verify reporting on rejection reasons to identify common issues
- Test that overly restrictive rejections are appealable
- Ensure consistency in document acceptance standards across administrators
