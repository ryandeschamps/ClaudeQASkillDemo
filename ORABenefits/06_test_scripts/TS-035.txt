Test Script ID: TS-035
Test Scenario: Administrator Process Manual Life Event
Priority: Critical
Preconditions:
- Employee reported life event outside timeliness window
- Late reporting request submitted with documentation
- Administrator has authority to approve late life events
- Manual processing workflow configured
- Exception approval criteria documented

GIVEN: An employee reported marriage life event 45 days late with reasonable explanation
WHEN: Benefits administrator manually processes and approves the late life event
THEN: Life event approved, enrollment window opened, and employee can complete enrollment

Test Steps:
1. Administrator receives late life event request in queue
2. Benefits administrator logs into Oracle Fusion HCM
3. Navigate to "Action Items" or "Pending Approvals"
4. Select "Late Life Event Requests" work queue
5. Review pending request: Employee reported marriage 45 days after event date
6. Review life event details:
   - Event type: Marriage
   - Event date: 2024-08-01
   - Reported date: 2024-09-15 (45 days late)
   - Reason: Extended honeymoon travel without system access
7. Review supporting documentation: Marriage certificate uploaded
8. Evaluate against exception criteria: reasonable cause, documentation provided
9. Click "Approve" or "Process Life Event"
10. Enter administrator approval notes: "Approved - reasonable cause with documentation"
11. Select coverage effective date: Marriage date (2024-08-01) or approval date
12. Open enrollment window for employee (30 days from approval)
13. Submit approval decision
14. System sends notification to employee: "Your late life event has been approved"
15. Employee logs in and sees active life event enrollment window
16. Employee adds spouse and completes benefit elections
17. Administrator reviews and finalizes enrollment
18. Verify retroactive premium adjustments if coverage backdated
19. Complete life event processing
20. Verify audit trail documents administrator approval and rationale

Expected Result:
- Administrator successfully accesses late life event requests
- Complete life event details visible including dates, reason, documentation
- Administrator evaluates request against exception criteria
- Life event approved with documented justification
- Approval notes captured in system
- Coverage effective date determined (event date or approval date)
- Enrollment window opened for employee (30 days from approval)
- Employee receives notification of approval with instructions
- Employee completes enrollment successfully
- Retroactive premium calculations if coverage backdated
- Audit trail captures: request, review, approval, administrator notes
- Exception properly documented for compliance

Test Data Reference:
- Variant IDs: V048, V049, V079 (Late Life Event processing)
- Any life event type reported outside standard window
- Requires administrator review and approval

Notes:
- Verify clear criteria for approving late life events
- Confirm documentation requirements consistently applied
- Test various scenarios: reasonable cause, hardship, extenuating circumstances
- Verify denial process if exception criteria not met
- Ensure employee notification for both approval and denial
- Test retroactive coverage and premium implications
- Confirm plan sponsor rules allow backdated coverage
- Verify compliance with plan document provisions on late enrollment
- Test that approved late events follow same workflow as timely events
- Ensure proper authorization limits for administrators
