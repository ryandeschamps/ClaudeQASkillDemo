Test Script ID: TS-044
Test Scenario: Upload Required Document
Priority: Critical
Preconditions:
- Employee enrolled dependent requiring documentation
- Action item created for required document
- Document upload capability functional in self-service portal
- Supported file formats configured (PDF, JPG, PNG)
- File size limits defined and communicated

GIVEN: An employee added a newborn to benefits and needs to upload birth certificate to activate coverage
WHEN: The employee uploads the required birth certificate document
THEN: Document successfully uploaded, action item updated, and submitted for administrator approval

Test Steps:
1. Employee completes enrollment adding newborn child to benefits
2. System creates action item: "Upload birth certificate for [child name]"
3. Employee logs into self-service portal
4. Dashboard displays notification: "You have 1 pending action item"
5. Click notification or navigate to "Action Items" section
6. Review action item details:
   - Type: Document Upload Required
   - Document: Birth certificate
   - For: Child name (Noah Jackson)
   - Due date: 30 days from birth date
   - Status: Pending
   - Note: "Coverage suspended until document approved"
7. Click "Upload Document" button
8. System displays file upload dialog
9. Review supported formats: PDF, JPG, PNG (max 10MB)
10. Click "Choose File" or drag and drop file
11. Select birth certificate file from computer
12. File validation: Check file type and size
13. Preview uploaded document to verify readability
14. Verify correct document selected
15. Add optional notes: "Official birth certificate from County Clerk"
16. Click "Submit" or "Upload Document"
17. System uploads file and confirms receipt
18. Action item status updates to "Document Submitted - Pending Approval"
19. Confirmation message: "Your document has been submitted for review"
20. Receive confirmation email with submission details
21. Can view uploaded document in self-service portal
22. Track approval status in action items section

Expected Result:
- Employee successfully accesses action items from dashboard
- Action item clearly explains what document needed and why
- Due date and coverage implications prominently displayed
- File upload interface user-friendly and intuitive
- Supported file formats and size limits clearly communicated
- File validation prevents unsupported formats or oversized files
- Document preview allows employee to verify correct file
- Upload completes successfully without errors
- Action item status updates to "Pending Approval"
- Employee receives confirmation of successful upload
- Confirmation email sent with next steps
- Employee can view uploaded document anytime
- Document routed to benefits administrator for approval
- Coverage remains suspended until approval (if applicable to plan)
