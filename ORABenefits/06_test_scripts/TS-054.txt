Test Script ID: TS-054
Test Scenario: Multiple Beneficiaries with Percentages
Priority: High
Preconditions:
- Employee enrolled in life insurance
- Beneficiary designation supports multiple beneficiaries
- Percentage allocation functionality enabled
- Validation rules for percentage totals configured
- Per stirpes vs. per capita options available

GIVEN: An employee wants to split life insurance proceeds among spouse and two children with specific percentages
WHEN: The employee designates multiple primary beneficiaries with percentage allocations
THEN: All beneficiaries successfully designated with percentages totaling 100%

Test Steps:
1. Employee logs into self-service portal
2. Navigate to "Beneficiaries" section
3. Click "Designate Beneficiaries" or "Update Beneficiaries"
4. Life insurance coverage: $100,000
5. Click "Add Primary Beneficiary"
6. Add first primary beneficiary:
   - Name: David Wright (Spouse)
   - Relationship: Spouse
   - Date of Birth: 1985-03-20
   - SSN: XXX-XX-XXXX
   - Percentage: 50%
7. Click "Save and Add Another Primary Beneficiary"
8. Add second primary beneficiary:
   - Name: Emily Wright (Child)
   - Relationship: Child
   - Date of Birth: 2010-06-15
   - SSN: XXX-XX-XXXX
   - Percentage: 25%
9. Click "Save and Add Another Primary Beneficiary"
10. Add third primary beneficiary:
    - Name: Michael Wright (Child)
    - Relationship: Child
    - Date of Birth: 2012-09-20
    - SSN: XXX-XX-XXXX
    - Percentage: 25%
11. System validates total percentage: 50% + 25% + 25% = 100% âœ“
12. Review primary beneficiary summary:
    - David Wright (Spouse) - 50% = $50,000
    - Emily Wright (Child) - 25% = $25,000
    - Michael Wright (Child) - 25% = $25,000
13. System displays per stirpes option for children:
    - "If a child beneficiary predeceases you, should their share go to their descendants?"
14. Select per stirpes option: Yes
15. Optionally add contingent beneficiaries
16. Add contingent beneficiary:
    - Name: Sarah Wright (Sister)
    - Percentage: 100% (receives if all primary beneficiaries predecease)
17. Review complete beneficiary designation:
    - Primary: 3 beneficiaries totaling 100%
    - Contingent: 1 beneficiary at 100%
18. Electronic signature acknowledgment
19. Type full name to sign
20. Submit beneficiary designation
21. Confirmation: "Multiple beneficiaries designated successfully"
22. Receive confirmation email with all beneficiary details and percentages

Expected Result:
- Employee can add multiple primary beneficiaries
- Percentage field for each beneficiary
- System calculates total percentage in real-time
- Validation prevents submission if total doesn't equal 100%
- System displays estimated proceeds for each beneficiary based on percentage
- Per stirpes option available for child beneficiaries
- Contingent beneficiaries can be added separately
- Contingent beneficiaries also must total 100%
- Complete beneficiary summary shows all designations with percentages
- Electronic signature captured
- All beneficiaries saved successfully
- Confirmation includes complete beneficiary breakdown with percentages and estimated amounts
- Beneficiary designation effective immediately

Test Data Reference:
- Variant IDs: V054 (Multiple beneficiaries scenario)
- Test Data: Row 54 (Employee with multiple children requiring percentage split)
- Common scenarios:
  - Spouse 50%, Children split remaining 50%
  - Equal split among multiple children
  - Unequal splits based on needs

Notes:
- Verify real-time percentage calculation and validation
- Confirm system prevents submission with incorrect total percentage
- Test per stirpes vs. per capita designation options
- Verify estimated proceeds calculation based on percentages
- Ensure minor children can be named with appropriate disclosures
- Test that percentages can be unequal (not required to be equal splits)
- Confirm contingent beneficiaries follow same percentage rules
- Verify all beneficiaries must total exactly 100% (not more, not less)
- Test decimal percentages if needed (e.g., 33.33%, 33.33%, 33.34%)
- Ensure beneficiary order doesn't matter (all paid simultaneously)
