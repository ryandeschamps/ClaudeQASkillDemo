Test Script ID: TS-022
Test Scenario: Late Life Event Reporting
Priority: High
Preconditions:
- Employee experienced qualifying life event outside reporting window
- Late life event reporting process configured
- Administrator review and approval workflow enabled
- Exception request documentation capability
- System tracks original life event date and reporting date

GIVEN: An employee (Kathleen Phillips) got married on 2024-02-10 but didn't report within 31-day window
WHEN: The employee reports marriage life event 45 days late and requests exception
THEN: Administrator reviews late report, approves exception, and processes enrollment with appropriate effective date

Test Steps:
1. Employee logs into self-service portal (kathleen.phillips@company.com) on 2024-04-01
2. Navigate to "Report Life Event" section
3. Select "Marriage" as life event type
4. Enter marriage date: 2024-02-10 (occurred 50 days ago)
5. System calculates reporting window expired (31 days from 2024-02-10)
6. System displays message: "This life event is outside the reporting window"
7. System prompts: "Would you like to submit for administrator review?"
8. Employee clicks "Submit Late Life Event Request"
9. System requests reason for late reporting
10. Employee enters explanation: "Honeymoon travel, didn't have access to system"
11. Employee uploads supporting documentation (marriage certificate)
12. Submit late life event request with reason
13. Request routed to benefits administrator queue
14. Benefits administrator logs in and reviews pending late life event requests
15. Administrator reviews: marriage date, reporting date, reason, documentation
16. Administrator evaluates exception criteria (reasonable cause, documentation provided)
17. Administrator approves late life event exception
18. Administrator notes: "Approved - reasonable cause, documentation provided"
19. System opens life event enrollment window for employee (30 days from approval)
20. Employee receives notification of approval and can now complete enrollment
21. Employee adds spouse and enrolls in coverage per standard marriage life event process

Expected Result:
- System identifies life event reported outside 31-day window
- Late reporting option presented to employee
- Employee submits late life event request with reason
- Request routed to benefits administrator for review
- Administrator can view all late report details (dates, reason, documentation)
- Administrator approves exception based on reasonable cause
- Life event enrollment window opened for employee (30 days from approval)
- Employee successfully completes enrollment adding spouse
- Coverage effective date determined by administrator (may be marriage date or approval date)
- Premium adjustments calculated from approved effective date
- Audit trail documents late report, reason, administrator decision
- Employee notified of exception approval and enrollment instructions

Notes:
- Verify system strictly enforces life event reporting windows
- Confirm late reporting requires administrator review and approval
- Test exception criteria (reasonable cause, documentation, timeliness of late report)
- Verify administrator can approve or deny late reports
- Ensure coverage effective date may differ based on approval (not automatically backdated)
- Test various late report scenarios (forgotten, travel, hospitalization, etc.)
- Confirm denial process and employee notification if exception not granted
- Verify audit trail captures all late report activities
- Test premium implications if coverage backdated vs. prospective
