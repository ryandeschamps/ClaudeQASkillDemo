Test Script ID: TS-069
Test Scenario: Bulk Update Benefits Groups via Workbook
Priority: Critical
Preconditions:
- Benefits administrator role with workbook upload privileges
- Workbook template for benefits group updates available
- Organizational change requiring benefits group reassignment
- HCM Data Loader or workbook upload functionality configured
- Validation rules for benefits group assignments

GIVEN: Company reorganization requires moving 150 employees from one benefits group to another
WHEN: Benefits administrator uploads workbook with benefits group changes
THEN: Benefits groups successfully updated in bulk for all employees with validation and confirmation

Test Steps:
1. Company announces organizational restructuring
2. 150 employees transferring from Division A (BG-001) to Division B (BG-002)
3. Effective date: 2025-01-01
4. Benefits administrator logs into Oracle Fusion HCM
5. Navigate to "Data Management" or "Workbook Uploads"
6. Click "Download Template" or "HCM Data Loader"
7. Select template: "Update Benefits Group Assignment"
8. Download template: Benefits_Group_Update_Template.xlsx
9. Open template in Excel
10. Template columns:
    - Person_Number (required)
    - Current_Benefits_Group (validation)
    - New_Benefits_Group (required)
    - Effective_Date (required)
    - Reason (required)
11. Populate template with 150 employee updates:
    - Person_Number: P100001, P100002, ... P100150
    - Current_Benefits_Group: BG-001
    - New_Benefits_Group: BG-002
    - Effective_Date: 2025-01-01
    - Reason: Division transfer - Organizational restructure
12. Save completed workbook: BG_Update_150_Employees.xlsx
13. Return to HCM Data Loader
14. Click "Upload Workbook" or "Import Data"
15. Select file: BG_Update_150_Employees.xlsx
16. Choose upload mode: "Validate and Process" (vs. "Rollback Mode" for testing)
17. Click "Upload" to begin processing
18. System validates workbook:
    - File format: Valid Excel ✓
    - Required columns present: ✓
    - Person numbers exist: ✓ (all 150 found)
    - Current benefits group matches: ✓
    - New benefits group exists: ✓ (BG-002 active)
    - Effective date valid: ✓ (future date)
19. Validation complete: 150 rows validated successfully, 0 errors
20. System displays validation summary
21. Click "Process Updates" to apply changes
22. System processes benefits group updates
23. Processing complete: 150 employees updated successfully
24. Review processing results:
    - Total rows: 150
    - Successful updates: 150
    - Failed updates: 0
25. Generate update report showing all changes
26. Verify sample employee records updated:
    - Employee P100001: BG-001 → BG-002 (effective 2025-01-01)
27. System sends notifications to affected employees
28. Employees notified of benefits group change and effective date

Expected Result:
- Workbook template easily accessible and downloadable
- Template includes all required fields with clear labels
- Template accepts 150+ employee rows efficiently
- File upload process user-friendly
- Validation comprehensive: file format, data integrity, referential integrity
- Validation identifies all errors before processing
- Validation summary clear and actionable
- Processing mode selection available (validate only vs. process)
- Bulk processing efficient for 150 employees
- All benefits group updates successful
- Processing results detailed with success/failure counts
- Updated records immediately visible in system
- Effective date respected (changes effective 2025-01-01)
- Affected employees notified of changes
- Audit trail captures bulk update with reason and date

Test Data Reference:
- Large population requiring bulk benefits group update
- Organizational changes: mergers, acquisitions, divestitures, reorganizations
- Requires efficient bulk data management

Notes:
- Verify workbook template current and comprehensive
- Confirm validation prevents invalid data from processing
- Test rollback mode for testing without committing changes
- Verify large file handling (500+ employees if needed)
- Ensure effective dating works correctly for future changes
- Test error handling and reporting for failed rows
- Confirm notification to affected employees
- Verify integration with benefits eligibility re-evaluation
- Test audit trail comprehensiveness
- Ensure performance acceptable for large populations
