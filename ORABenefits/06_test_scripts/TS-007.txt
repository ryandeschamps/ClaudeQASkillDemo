Test Script ID: TS-007
Test Scenario: New Hire with Court Order (QMCSO)
Priority: Medium
Preconditions:
- QMCSO (Qualified Medical Child Support Order) processing capability enabled
- Benefits administrator trained on court order requirements
- New hire has valid court order requiring dependent coverage
- Court order document upload and verification process functional
- System allows enrollment of non-custodial dependent children

GIVEN: A new employee has a court order requiring them to provide health coverage for a child not in their custody
WHEN: Benefits administrator enrolls new hire and adds court-ordered dependent per QMCSO requirements
THEN: The designated dependent is successfully enrolled per court order specifications and system flags enrollment as QMCSO-related

Test Steps:
1. Log into Oracle Fusion HCM as Benefits Administrator
2. Navigate to Benefits Administrator work area
3. Receive court order documentation from new hire or HR
4. Review court order to verify it meets QMCSO requirements (specifies child, coverage type, duration)
5. Search for and select new hire employee in system
6. Initiate administrator-assisted enrollment for new hire
7. Navigate to "Add Dependent" section
8. Add court-ordered dependent with required information from court order
9. Mark dependent as "Court Ordered" or select QMCSO indicator checkbox
10. Upload court order document to employee's document repository
11. Tag document with "QMCSO" or "Court Order" classification
12. Proceed to benefits enrollment section
13. Enroll employee and court-ordered dependent in medical/dental plan as specified in order
14. Verify system prevents removal of court-ordered dependent
15. Add administrator notes documenting QMCSO compliance
16. Submit enrollment elections
17. Verify enrollment confirmation includes court-ordered dependent
18. Create reminder task for annual QMCSO review and renewal
19. Send enrollment confirmation to both employee and court/custodial parent if required

Expected Result:
- Court-ordered dependent successfully added with QMCSO indicator
- Dependent enrolled in required health coverage per court order specifications
- System flags dependent as "Court Ordered" preventing unauthorized removal
- Court order document stored and linked to dependent record
- Premium correctly calculated including court-ordered dependent
- Coverage effective date complies with court order requirements
- Administrator notes document QMCSO compliance
- System generates reminder for periodic QMCSO review
- Audit trail clearly shows QMCSO-related enrollment

Test Data Reference:
- Variant IDs: V006 (Administrator-Assisted, Document Status scenarios)
- Test Data: Adapt from Row 7 (P100006, Patricia Martinez)
  - Enrollment_Method: Administrator-Assisted
  - Document_Type: Court Order (QMCSO)
  - Coverage_Level: Employee+Children
  - Action_Item_Status: Pending or Completed based on document review

Notes:
- QMCSO compliance is legally mandated - verify system prevents non-compliance
- Confirm system restricts employee's ability to remove court-ordered dependent
- Verify plan administrator notification if employee terminates employment
- Test that dependent aging out does not automatically remove if court order still valid
- Ensure system alerts administrator if court order expiration date approaching
- Verify premium allocation complies with court order specifications
- Test COBRA rights for court-ordered dependent upon employee termination
- Confirm proper notification to custodial parent as required by court order
