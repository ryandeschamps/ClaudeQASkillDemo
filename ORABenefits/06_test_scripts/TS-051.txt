Test Script ID: TS-051
Test Scenario: Add Dependent Contact
Priority: Critical
Preconditions:
- Employee has access to self-service portal
- Dependent management functionality enabled
- Contact information fields configured
- Relationship types defined in system
- Validation rules for required fields configured

GIVEN: An employee needs to add a new dependent (spouse, child, or other qualified dependent) for benefits coverage
WHEN: The employee adds dependent with complete personal information
THEN: Dependent successfully added as contact and available for benefit enrollment

Test Steps:
1. Employee logs into self-service portal
2. Navigate to "Benefits" or "Personal Information" section
3. Click "Dependents" or "Contacts and Dependents"
4. View existing dependents list (may be empty for new employees)
5. Click "Add Dependent" or "Add New Contact"
6. System displays dependent information form
7. Select relationship type from dropdown:
   - Spouse
   - Domestic Partner
   - Child
   - Stepchild
   - Adopted Child
   - Other Dependent
8. Select: "Child"
9. Enter dependent's personal information:
   - First Name: Emma
   - Middle Name: Grace
   - Last Name: Anderson
   - Date of Birth: 2010-05-15
   - Gender: Female
   - Social Security Number: XXX-XX-XXXX
10. Enter contact information:
   - Same address as employee (checkbox)
   - Phone: (555) 123-4567
   - Email: (optional for minor children)
11. Enter additional required information:
   - Student status: Yes/No
   - Disabled dependent: Yes/No
   - Tobacco user: Yes/No (if required for rates)
12. Review validation requirements:
   - All required fields completed
   - Date of birth in valid format and range
   - SSN format valid (if required)
   - Age appropriate for selected relationship
13. Click "Save" or "Add Dependent"
14. System validates all entries
15. System saves dependent contact record
16. Confirmation message: "Dependent Emma Anderson added successfully"
17. Dependent appears in dependent list
18. Dependent now available for benefit enrollment selections
19. Action item may be created: "Upload birth certificate for Emma Anderson"
20. Employee can proceed to enroll dependent in benefits

Expected Result:
- Employee successfully accesses dependent management functionality
- Add dependent form displays all necessary fields
- Relationship types comprehensive and clearly labeled
- Form includes all required personal and contact information fields
- Validation ensures required fields completed correctly
- SSN format and date of birth validations work properly
- Age validation appropriate for relationship type (e.g., child must be under 26)
- Dependent contact successfully saved to system
- Confirmation message confirms successful addition
- Dependent appears in dependent list immediately
- Dependent available for benefit enrollment
- Appropriate action items created for required documents
- Employee can add multiple dependents as needed

Test Data Reference:
- Variant IDs: All variants with Number_Of_Dependents > 0
- Test Data: Examples showing dependent information
  - Row 3 (P100002, Maria Rodriguez): 1 dependent (spouse)
  - Row 4 (P100003, Michael Chen): 2 dependents (children)
  - Row 5 (P100004, Jennifer Williams): 3 dependents (spouse + children)

Notes:
- Verify all standard relationship types available
- Confirm required vs. optional fields clearly marked
- Test SSN validation and format requirements
- Verify date of birth validation (reasonable ranges)
- Test age validation for dependent eligibility
- Ensure duplicate detection (same dependent not added twice)
- Confirm student status tracking for dependent eligibility extension
- Verify disabled dependent designation for age limit exceptions
- Test that tobacco user status captured if affects rates
- Ensure contact information can be same as or different from employee
