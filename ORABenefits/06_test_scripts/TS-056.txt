Test Script ID: TS-056
Test Scenario: Beneficiary Organization (Trust)
Priority: Medium
Preconditions:
- Employee enrolled in life insurance
- Trust as beneficiary option configured
- EIN/TIN validation enabled for organizational beneficiaries
- Trust documentation upload capability
- Legal entity beneficiary designation supported

GIVEN: An employee has established a trust for estate planning and wants to designate the trust as life insurance beneficiary
WHEN: The employee designates their trust as beneficiary instead of individual persons
THEN: Trust successfully designated as beneficiary with proper identification and documentation

Test Steps:
1. Employee logs into self-service portal
2. Navigate to "Beneficiaries" section
3. Click "Add Primary Beneficiary"
4. Select beneficiary type: "Organization/Trust" (vs. "Individual Person")
5. System displays organizational beneficiary form with different fields
6. Enter trust information:
   - Organization Name: "The Smith Family Trust"
   - Organization Type: "Trust"
   - Tax ID Number (EIN): XX-XXXXXXX
   - Date Established: 2020-05-15
7. Enter trustee information:
   - Trustee Name: John Smith
   - Trustee Phone: (555) 345-6789
   - Trustee Email: john.smith@email.com
8. Enter trust address:
   - Street Address: 789 Estate Lane
   - City: Trust City
   - State: NY
   - ZIP: 10001
9. Percentage allocation: 100%
10. Upload trust documentation:
    - Trust agreement or certificate
    - Click "Upload Document"
    - Select trust document file
    - Submit upload
11. Review trust beneficiary designation:
    - Beneficiary: The Smith Family Trust (EIN: XX-XXXXXXX)
    - Trustee: John Smith
    - Percentage: 100%
    - Documentation: Trust Agreement (uploaded)
12. System validates:
    - EIN format correct
    - Required trust information complete
    - Trust documentation uploaded
13. Optionally add contingent beneficiaries (individuals or another trust)
14. Add contingent: Estate of Employee (100%)
15. Review complete beneficiary designation:
    - Primary: The Smith Family Trust - 100%
    - Contingent: Estate of [Employee Name] - 100%
16. System displays notice about trust beneficiaries:
    - "Trust beneficiaries require additional documentation for claims"
    - "Ensure trustee has current contact information"
17. Electronic signature acknowledgment
18. Type full name to sign
19. Submit trust beneficiary designation
20. Confirmation: "Trust designated as beneficiary successfully"
21. Receive confirmation with trust details and trustee contact

Expected Result:
- Employee can select "Organization/Trust" as beneficiary type
- Form adapts to show organizational/trust-specific fields
- EIN/TIN field instead of SSN for trust identification
- Trust name, type, and establishment date captured
- Trustee contact information required and captured
- Trust address captured (may differ from employee address)
- Percentage allocation works same as individual beneficiaries
- Trust documentation can be uploaded for reference
- System validates EIN format and required fields
- Trust successfully designated as primary beneficiary at 100%
- Contingent beneficiaries can still be designated (backup to trust)
- System provides notice about additional trust documentation for claims
- Trust designation confirmed and saved
- Confirmation includes trust name, EIN, trustee contact
- Trust designation effective immediately

Test Data Reference:
- Trust as beneficiary scenarios
- Estate planning situations
- High net worth employee scenarios
- Employees with complex estate plans

Notes:
- Verify trust option available and clearly labeled
- Confirm EIN validation appropriate for tax identification numbers
- Test that trustee information required and captured
- Verify trust documentation upload capability
- Ensure trust address can be different from employee
- Test that trust can be primary, contingent, or partial beneficiary
- Confirm percentage rules same for trust as individual beneficiaries
- Verify claims process documentation clearly communicated
- Test that trust beneficiary requires additional verification during claims
- Ensure trust designation can be updated or changed
